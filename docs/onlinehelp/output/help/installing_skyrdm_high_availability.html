Installing SkyDRM RMS on multiple servers for high availability

This section describes the SkyDRM RMS installation procedure on a Linux server cluster, using Docker images. The server clustering uses Docker swarm. A Docker swarm is a group of machines that are running Docker and form a cluster.

Upon completing the setup, SkyDRM RMS creates three property files.

Important: Ensure that you type the correct information, especially passwords because during installation, SkyDRM encrypts the passwords. In the resulting properties files, the passwords are masked and you can only change them by re-running the setup script.
  1. Log in to your Linux server as a user with sudo privileges.
  2. Navigate to the folder containing the Docker images.
  3. Run the command to load the Docker images. 
    docker load -i <name of the Docker image>
    Repeat this step to load all Docker images, in all servers, in the following sequence.
    Important: If you are using an external database, you do not need to load the database image.
    1. RightsManagementServer-DB-DockerImage-10.*.tar
    2. RightsManagementServer-RMS-DockerImage-10.*.tar
    3. RightsManagementServer-Router-DockerImage-10.*.tar
    4. RightsManagementServer-Viewer-DockerImage-10.*.tar
    5. RightsManagementServer-CacheServer-DockerImage-10.*.tar
    6. RightsManagementServer-MessageQueue-DockerImage-10.*.tar
  4. Check that images are loaded by running the following command. 
    docker images
    A list of the Docker images appear: the SkyDRM RMS server, router, and viewer images, and the database image if you are using the embedded PostgreSQL image.
  5. If this is the first time you are deploying a Docker stack, initialize the Docker swarm by running the following command. 
    docker swarm init --advertise-addr <IP address of the server>
    Docker prints a few commands execute. Ignore these commands.
  6. Get the manager token by running the following command. 
    $sudo docker swarm join-token manager
  7. Copy the token value.
  8. In the other servers, run the following command for the servers to join the cluster. 
    $sudo docker swarm join --token <token><manager IP address>
  9. Verify that the Docker swarm cluster is up and running by running the following command. 
    $ sudo docker node ls
  10. Extract the SkyDRM setup files by running the following command. 
    unzip SkyDRM-setup-*.zip
  11. Navigate to /SkyDRM-setup-<xxx>/setup.
  12. Check if you have permission to run the setup script by running the following command. 
    ls -l
    Note: If you do not have permission to run the setup script, the Linux host system owner can run the following command to give read, write, execute permission to the SkyDRM administrator.
    chmod 500 setup.sh
  13. Run the setup script setup.sh by running the following command. 
    sudo sh setup.sh
    The script asks if you want to specify your own data directory. All your configuration files go to this location, and it becomes the SkyDRM RMS installation path.
  14. Type n to use the default data directory.
    The Configure IdP or identity provider setup steps appear. Here, you can configure LDAP, SAML, Facebook, Google, or native SkyDRM RMS IdP as identity providers.
  15. Type the following options to configure the IdP.
    • 1 to configure LDAP.
    • 2 to configure SAML.
    • 3 to configure Facebook.
    • 4 to configure Google.
    • 5 to use SkyDRM RMS IdP.
  16. Follow the on-screen steps to configure each IdP.
  17. After you have configured all the necessary IdPs, type n to proceed to the next configuration set.
    Alternatively, you can type y to configure other IdPs.
    The Configure default storage provider setup steps appear. Here, you can select either Amazon S3, OneDrive for Business, or local drive as your storage provider. You can only select one default storage provider.
  18. Type the following options to configure the default storage provider.
    • 1 to configure Amazon S3.
    • 2 to configure OneDrive for Business.
    • 3 to configure local drive.
    Important: You must configure a default storage provider, even if you intend to use SkyDRM to protect files in other types of storage, for example, a network file share location. SkyDRM uses the default storage to store and update configuration informatyion and other runtime information.
  19. Follow the onscreen steps to configure each default storage provider.
    Note: Your enterprise system administrators are responsible for the management of local storage, including its availability, backup plans, recovery plans, consistency, and so on.
  20. After you have configured the default storage provider, proceed to the next configuration set.
    The Configure SMTP server setup steps appear. Here, you can configure a mail server which sends email notifications to people you choose to share the protected files with. You can only select one SMTP server.
  21. Type the SMTP server name.
  22. Type the SMTP port.
  23. When asked if SSL is enabled for SMTP, type y or n.
  24. Type the SMTP user name.
  25. Type the SMTP password.
  26. After you have configured the SMTP server, proceed to the next configuration set.
    The Configure database setup steps appear. Here, you can configure your new SkyDRM RMS database.
  27. When asked if you want to use the embedded database, type y or n, depending on the database you are using.
    If you typed n, a list of the available database types appear. Type the number associated with one of the following databases to configure it.
    • 1 to configure PostgreSQL database.
    • 2 to configure Microsoft SQL Server database.
    • 3 to configure Oracle database.
    The Admin configuration setup steps appear. Here, you can configure the host name or IP address for your SkyDRM RMS installation.
  28. Type the hostname of the server that will host SkyDRM RMS.
    Note: The hostname must always be the fully-qualified domain name (FQDN), and never the IP address.
  29. Type the email address of the SkyDRM RMS administrator.
    The Control Center Policy Server setup steps appear. Here, you can configure the Control Center server for your SkyDRM RMS installation.
  30. Type the ICENet URL.
    For example: https://icenet.nextlabs.com:8443.
  31. Type the Control Center administrator user name.
    For example: Administrator.
  32. Type the Control Center administrator password.
  33. Type the Control Center console URL.
    For example: https://console.nextlabs.com
    The Viewer configuration setup steps appear. Here, you can choose to have one or multiple SkyDRM RMS viewers.
  34. When asked if you want to have multiple viewers, type one of the following options.
    • n for No, if you only want to have one viewer.
    • y for Yes, if you want to have multiple viewers.
  35. When asked if you want the cache used by SkyDRM RMS and Viewer to be remotely hosted, type n for No.
  36. Type the RabbitMQ host URL.
    This is the URL that SkyDRM RMS uses to send and receive notifications. SkyDRM RMS uses the RabbitMQ message broker to provide the message queue service.
  37. Type the RabbitMQ host port number.
  38. Type the RabbitMQ API host.
  39. Type the RabbitMQ API port number.
    This the API that SkyDRM RMS uses to create the RabbitMQ account.
    If you are using a Load Balancer, it must be configured to route the requests to the internal RabbitMQ server.
  40. Press Enter on your keyboard.
    A message appears, confirming the successful configuration. You can find the resulting property files in ${RMS_DATA_DIR}shared/conf/.
  41. Deploy the Docker stack by running the following command. 
    docker stack deploy --compose-file <composefilename> <stack name>

    For example:

    docker stack deploy --compose-file docker-compose-10-0129-201807100159.yml skydrm
  42. Check that you can access the SkyDRM RMS site.
    The script creates the URL to access SkyDRM RMS from a browser. For example: https://<hostname>:8444/rms
  43. Copy the viewer packages for the viewers you want to install to /opt/nextlabs/rms/shared/viewerPackages.
  44. Restart the Docker server by running the following command. 
    systemctl restart docker
