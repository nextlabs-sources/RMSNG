Integrating AD FS

Active Directory Federation Services (AD FS) is a Microsoft software component that provides users with single sign-on access to systems and applications located across organizational boundaries. SkyDRM RMS can integrate with AD FS, enabling you to log in to SkyDRM RMS using AD FS credentials. This topic describes how to add a relying party trust in AD FS, which binds AD FS to SkyDRM RMS. You can then add the AD FS information to admin.properties file which enables login with AD FS credentials.
  1. In your AD FS console, select Trust Relationships > Relying Party Trusts.
    The Action pane appears.
  2. Click Add Relying Party Trust.
    The Add Relying Party Trust Wizard appears.
  3. Click Start.
    The Select Data Source pane appears.
  4. Click the Enter data about the relying party manually radio button.
  5. Click Next.
    The Specify Display Name pane appears.
  6. Type the display name.
  7. Click Next.
    The Choose Profile pane appears.
  8. Click the AD FS Profile radio button.
  9. Click Next.
    The Configure Certificate pane appears.
  10. Click Next.
    The Configure URL pane appears.
  11. Select the Enable support for the SAML 2.0 WebSSO protocol check box.
  12. In the Relying party SAML 2.0 SSO service URL box, type https://<hostname>:8444/rms/IdpManager/SamlAuth/SamlAuthFinish.
  13. Click Next.
    The Configure Identifiers pane appears.
  14. In the Relying party trust identifier box, type https://<hostname>:8444/rms/IdpManager/SamlAuth/SamlMetadata.
  15. Click Add.
  16. Click Next.
    The Configure Multi-factor Authentication Now? pane appears.
  17. Click the I do not want to configure multi-factor authentication settings for this relying party trust at this time radio button.
  18. Click Next.
    The Choose Issuance Authorization Rules pane appears.
  19. Click the Permit all users to access this relying party radio button.
  20. Click Next.
    The Ready to Add Trust pane appears.
  21. Click Next.
    The Finish pane appears.
  22. Click Close.
    The Edit Claim Rules dialog box appears.
  23. Click Add Rule.
    The Add Transform Claim Rule Wizard dialog box appears.
  24. In the Choose Rule Type pane, from the claim rule template drop-down list, select Send LDAP Attributes as Claims.
  25. Click Next.
    The Configure Claim Rule pane appears.
  26. From the Attribute store drop-down list, select Active Directory.
  27. In the Mapping of LDAP attributes to outgoing claim types table, type the following mappings.
    LDAP AttributeOutgoing Claim Type
    E-Mail-Addresses
    Important: You must configure this attribute first.
    		 Name ID
    Important: You must type this value. DO NOT select it from the drop-down list.
    E-Mail-Addresses email
    Important: You must type this value. DO NOT select it from the drop-down list.
    Department department
    Important: You must type this value. DO NOT select it from the drop-down list.
    Company company
    Important: You must type this value. DO NOT select it from the drop-down list.
  28. Click Finish.
    The Edit Claim Rules dialog box appears.
  29. Click Apply.
  30. Click OK.
    The AD FS console appears.
  31. Navigate to AD FS > Service > Certificates.
  32. In the Certificates pane, under Token-signing, double-click the certificate.
    The Certificate dialog box appears.
  33. Click the Details tab.
  34. Click Copy to File.
    The Certificate Export Wizard dialog box appears.
  35. Click Next.
    The Export Private Key pane appears, asking if you want to export the private key with the certificate.
  36. Click the No, do not export the private key.
    The Export File Format pane appears, asking you to select the file format you want to use.
  37. Click the Base-64 encoded X.509 (.CER) radio button.
  38. Click Next.
    The File to Export pane appears.
  39. Click Browse to select the folder you want to save the certificate file to.
  40. Type a file name for the certificate file.
  41. Click Save.
  42. Click Next.
    The Certificate Export Completion information appears.
  43. Click Finish.
  44. Navigate to the certificate file, right-click and select Open with > Notepad.
  45. Copy and save the AD FS-related parameters for use during SkyDRM RMS installation.
    1. Copy the relying party SAML 2.0 SSO Service URL to the spAcsUrl.
    2. Copy the relying party trust identifier URL to the spEntityId.
    3. Copy the X509 Certificate to idpX509Cert.
    Sample configuration
    web.idp.saml.2.attributes={\"name":"ADFS",\
				"spEntityId":"https://<hostname>:8444/rms/IdpManager/SamlAuth/SamlMetadata",\
				"spAcsUrl":"https://<hostname>:8444/rms/IdpManager/SamlAuth/SamlAuthFinish",\
				"idpEntityId":"http://<AD server hostname>/adfs/services/trust",\
				"idpSsoUrl":"https://<AD server hostname>/adfs/ls/",\"spNameIdFormat":"",\"signAlgo":"",\"authNContext":"",\"buttonText":"ADFS",\
				"idpX509Cert":"MIIC+jCCAeKgAwIBAgIQa3x9rx0q5LFOWqXl211"\}
