Creating a policy

You must map user attributes and define document classification before attempting to create a policy. If you do not map the user attributes, the people components are not available to configure in the policy. If you do not define the document classification, you cannot create a policy.
Note: A policy applies to a file in a project, and determines who can access the file. However, and ABAC policy applies to an entire project, and determines who can access the project.
  1. Navigate to the administrator login page https://<hostname>:8444/rms/loginAdmin.
  2. Type your user name.
  3. Type your password.
  4. Click Log in.
    The Super Admin Dashboard page - System Settings tab appears.
  5. On the navigation bar, click Domain policies.
  6. Click Add policy.
    The New policy page appears.
  7. Type the name of the policy.
  8. Type a description for the policy.
  9. In the Resources pane, define the resource you want to protect, and their conditions.
    1. Select a condition from the drop-down list.
    2. Select a category name from the drop-down list.
      Note: File Extension is not supported.
    3. Select an operator from the drop-down list.
    4. Select a classification name from the drop-down list.
    You can add any of the following options.
    • An AND condition.
    • An OR block.
    • An AND block.
  10. In the People pane, define the people who can access the resources.
    1. From the Selected condition drop-down list, select a condition.
    2. Select an attribute from the drop-down list.
    3. Select an operator from the drop-down list.
    4. Define the values for the attribute.
      For example, for the age group attribute, you can define the attribute value as an age range between 20 to 40.
    You can add any of the following options.
    • An AND condition.
    • An OR block.
    • An AND block.
  11. In the Protect using digital rights pane, click the corresponding slider to select default rights protection options.
    • View: Enables users to view a rights-protected file. This is selected by default and cannot be disabled.
    • Print: Enables users to print a rights-protected file.
    • Download: Enables users to download a rights-protected file.
  12. Click one of the following options.
    • Save to save the new policy without deploying it.
    • Save & deploy to save the new policy and deploy it immediately.
