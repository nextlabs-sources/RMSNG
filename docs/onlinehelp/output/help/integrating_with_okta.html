Integrating Okta

Okta provides a service to secure identity management, and enables single sign-on to any application in the cloud, on-premises, or on a mobile device. SkyDRM RMS can integrate with Okta, enabling you to log in to SkyDRM RMS using Okta credentials. This topic describes how to create an Okta app, which binds Okta to SkyDRM RMS. You can then add the Okta information to admin.properties file which enables login with Okta credentials.
  1. Log in to Okta as an administrator.
  2. Click Admin.
  3. On the Admin Dashboard page, select Developer Console > Classic UI.
  4. In the Shortcuts menu, click Add Applications.
  5. On the Add Application page, click Create New App.
  6. In the Create a New Application Integration dialog box, click the SAML 2.0 radio button.
  7. Click Create.
  8. On the Create SAML Integration - General Settings page, type the App name.
  9. Click Next.
  10. On the Create SAML Integration - Configure SAML page, type the SkyDRM RMS Single sign on URL. For example, https://<hostname>:8444/rms/IdpManager/SamlAuth/SamlAuthFinish.
  11. Type the Audience URI (SP Entity ID). For example, https://<hostname>:8444/rms/IdpManager/SamlAuth/SamlMetadata.
  12. Define the attribute statement mappings to SkyDRM RMS attributes.
  13. Click Next.
  14. On the Create SAML Integration - Feedback page, click the I'm an Okta customer adding an internal app radio button.
  15. Select the This is an internal app that we have created check box.
  16. Click Finish.
    The new application settings page appears.
  17. Click the Assignments tab.
  18. Click Assign and then one of the following buttons.
    • Assign to People to assign to one person or more, one at a time.
    • Assign to Groups to assign to one or more groups.
  19. Click Assign next to the person or group.
  20. Click Save and go Back.
    Repeat this steps as many times as necessary, to add more people.
  21. Click Done.
  22. Copy and save the Okta-related parameters for use during SkyDRM RMS installation.
    1. Click the General tab.
    2. Copy the Single Sign On URL to the spAcsUrl.
    3. Copy the Audience Restriction to the spEntityId.
    4. Click the Sign On tab.
    5. Click View Setup Instructions.
    6. Copy the Identity Provider Single Sign-On URL to idpSsoUrl.
    7. Copy the Identity Provider Issuer to idpEntityId.
    8. Copy the X509 Certificate to idpX509Cert.
    Sample configuration
    web.idp.saml.1.attributes={\
                buttonText: "okta login", \
                spEntityId: "https://<hostname>:8444/rms/IdpManager/SamlAuth/SamlMetadata",\
                spAcsUrl: "https://<hostname>:8444/rms/IdpManager/SamlAuth/SamlAuthFinish", \
                idpEntityId: "http://www.okta.com/exkeof1q34wk4XuWV0h7",   \
                idpSsoUrl: "https://<yourcompany>.oktapreview.com/app/nextlabsdev189160_rmscentos1111_1/exkeof1lllwk4XuWV0h7/sso/saml", \
                idpX509Cert:"MIIDpDCCllllAwIBAgIGAWBOZ9peMA0GCSqGSIb3DQE"\
}
