Container-based deployment

You can install SkyDRM RMS using Docker, a container-based deployment platform. The SkyDRM RMS Docker image is a package that includes applications and their dependencies together in an isolated container, making them portable to any infrastructure. With a Docker image, you can ensure that the applications work consistently in any environment. A Docker image is lightweight and easily transportable, and can be placed anywhere for easy access.

The SkyDRM RMS installation package comes with the following components and Docker images.
  • Docker images
    • SkyDRM RMS database Docker image
    • SkyDRM RMS server Docker image
    • SkyDRM RMS router Docker image
    • SkyDRM RMS viewer Docker image
    • SkyDRM RMS notification system Docker image
    • SkyDRM RMS distributed cache server Docker image
  • Docker compose file to start the Docker service. The Docker compose file name is docker-compose-xxxx.yml.

    This file is included in the archive SkyDRM-setup-10-*.zip.

  • setup.sh, the shell script to provide the configure the values required by SkyDRM RMS installation. The script builds three property files which provide different parameters to SkyDRM RMS:
    • admin.properties file
    • viewer.properties file
    • router.properties file
