Policy creation in SkyDRM RMS

In SkyDRM, you can create access policies that are fully customizable to your needs. These policies apply to a specific project. You can define the resources you want to protect, the people who can access those resources, and the type of protection you want to apply.

NextLabs SkyDRM RMS assumes that users are granted rights to perform different actions (View, Print, etc.). For example, if you have not granted View rights for a document to a set of users, SkyDRM RMS denies them from viewing it.

In SkyDRM RMS, you can only create policies to be used for files shared in a SkyDRM project. The policies appear to project members as company-defined rights that you can apply to your files.

You can create policies in SkyDRM RMS, which are then sent to NextLabs Control Center. No policies are stored in SkyDRM RMS.
