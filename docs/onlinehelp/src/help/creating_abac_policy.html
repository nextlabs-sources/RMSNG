Creating an attribute-based access control policy

An Attribute-based access control (ABAC) policy uses attributes, or properties, such as information about the user or the environment, in authorization requests for access to assets. Access is allowed or denied based on policies.

This section describes how to create and ABAC policy.

Note: A policy applies to a file in a project, and determines who can access the file. However, and ABAC policy applies to an entire project, and determines who can access the project.
  1. Navigate to the administrator login page https://<hostname>:8444/rms/loginAdmin.
  2. Type your user name.
  3. Type your password.
  4. Click Log in.
    The Super Admin Dashboard page - System Settings tab appears.
  5. On the navigation bar, click Tenant management.
  6. Click Configure next to the name of the default tenant.
  7. In the Project settings - Project policies pane, click Create or manage project tags.
  8. Click Add project policy.
    The New project policy page appears.
  9. In the Name of the policy box, type a new project name.
  10. In the Project tags pane, click to select a project tag.
    You can select multiple project tags.
  11. In the People pane, define the people who can access the resources.
    1. From the Selected condition drop-down list, select a condition.
    2. Select an attribute from the drop-down list.
    3. Select an operator from the drop-down list.
    4. Define the values for the attribute.
      For example, for the age group attribute, you can define the attribute value as an age range between 20 to 40.
    You can add any of the following options.
    • An AND condition.
    • An OR block.
    • An AND block.
  12. Click one of the following options.
    • Save to save the new policy without deploying it.
    • Save & deploy to save the new policy and deploy it immediately.
